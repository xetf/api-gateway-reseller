# 模型调度与分发系统

这套系统决定一次 API 调用应该走哪个模型池、哪个上游渠道、哪个上游 Key。核心目标是：同一个 IP 尽量稳定，慢了会解绑，失败会惩罚，健康检测负责恢复和测速。

## 1. 粘性规则

某个 IP 第一次成功调用某个模型后，会粘住本次命中的上游渠道和上游 Key。默认粘性时间是 600 秒。后续同 IP、同模型再次成功调用，会刷新这个 600 秒窗口。

后台可配置：

- 是否开启粘性规则
- 粘性时长

## 2. 慢定义规则

一次调用满足任一条件就算慢：

- 首 token 时间超过 15000 毫秒
- 总时长超过 45000 毫秒

所有 compact 相关请求不计入慢定义。慢定义只服务于粘性解绑；如果粘性解绑关闭，慢定义不会产生效果。

后台可配置：

- 首 token 慢阈值
- 总时长慢阈值

## 3. 粘性解绑规则

某个 IP 已经命中粘性后，如果连续慢调用达到默认 3 次，就清除这个 IP 对该模型的粘性。下次请求会重新走标准选择。

如果粘性规则关闭，解绑规则自动无效。如果粘性规则开启，解绑规则也可以单独关闭，表示不因慢调用解绑。

后台可配置：

- 是否开启粘性解绑
- 连续慢几次解绑

## 4. 标准选择规则

当一个 IP 没有粘性，或者粘性已经被解绑，就进入标准选择规则。标准选择永远开启，无法关闭。

标准选择会先根据模型和等级找到可选池，再按熵值选择最合适的上游渠道和 Key。

## 5. 可选池规则

模型池按模型和等级组织。调用者只能在自己等级允许的模型池里选择上游渠道。

后台模型池页面按：

- 模型
- 等级
- 可调用区
- 不可调用区

来展示。

## 6. 等级规则

等级可以来自 IP、API Key、用户，优先级是：

1. IP 等级
2. API Key 等级
3. 用户等级
4. standard 默认等级

IP 等级支持单 IP 和 IPv4 CIDR，例如 `1.2.3.4` 或 `1.2.3.0/24`。

等级预设在后台“调度与分发”页面维护；每个用户的等级在“用户管理”里直接设置。套餐、租户、月结不再参与等级判断。

## 7. 负载均衡规则

进入标准选择后，系统会先选熵值最低的上游渠道，再选该上游里熵值最低的 Key。负载均衡永远开启，无法关闭。

## 8. 熵值规则

熵值越低，越容易被选中。

上游渠道熵值来自：

- 速度排名熵增：健康检测测出的首 token 平均时间越快，排名越靠前，熵增越小。
- 粘性 IP 数熵增：越多 IP 粘住这个渠道，熵值越高。

上游 Key 熵值来自：

- 粘性 IP 数熵增：越多 IP 粘住这个 Key，熵值越高。

当前计算口径：

- 渠道熵值 = `速度排名熵增 + 渠道粘性 IP 数 * stickyHitPenalty`
- 速度排名熵增 = `(速度排名 - 1) * speedRankPenalty`
- Key 熵值 = `Key 粘性 IP 数 * stickyHitPenalty`

速度排名来自健康检测记录的首 token 平均时间，从小到大排序。第 1 名速度排名熵增为 0。

后台可配置：

- 速度排名熵增
- 粘性数量熵增

## 9. 惩罚规则

同一个 IP 连续调用同一个上游渠道失败默认 2 次后，这个渠道进入惩罚中。不同 Key 也算同一个上游渠道。

后台可配置：

- 是否开启惩罚规则
- 连续失败几次触发惩罚
- 惩罚时长

## 10. 健康检测规则

健康检测默认每 30 秒执行一次，无法关闭。检测时会对该上游渠道下所有 active Key 同时发起小请求。

- 全部 Key 成功：渠道保持可调用
- 任一 Key 失败：渠道进入不可用区

检测开始后会显示检测中。原本可调用的渠道检测中仍可调用；原本不可调用的渠道检测中仍不可调用。

后台可配置：

- 健康检测间隔

## 11. 状态规则

渠道有三个核心状态：

- 可调用：参与调度，显示在可调用区。
- 惩罚中：不参与调度，显示在不可调用区；惩罚到期后触发健康检测。
- 检测中：只是过程状态，不单独改变是否可调用。

状态规则永远开启，无法关闭。

## 12. 强制可用规则

管理员可以把某个模型池渠道设置为强制可用。强制可用会无视惩罚、不可用和健康失败，始终参与调度。

强制可用渠道仍会参与健康检测，检测结果只用于更新速度排名和错误信息。

后台可配置：

- 是否允许显示/使用强制可用按钮

## 后台配置字段

这些字段保存在 `SystemSetting` 的 `model_dispatch_settings` 里。

| 字段 | 默认值 | 白话说明 |
| --- | ---: | --- |
| `stickyEnabled` | `true` | 是否开启 IP + 模型粘性。关闭后不读写粘性，并清理旧粘性。 |
| `stickyTtlSeconds` | `600` | 粘住多久；成功调用会刷新这个时间窗口。 |
| `stickySlowUnbindEnabled` | `true` | 是否允许“连续慢”解绑粘性。粘性关闭时这个规则自动无效。 |
| `slowFirstTokenMs` | `15000` | 首 token 超过这个时间，本次调用算慢。 |
| `slowTotalLatencyMs` | `45000` | 总耗时超过这个时间，本次调用算慢。 |
| `slowUnbindThreshold` | `3` | 连续慢多少次后解绑该 IP 对该模型的粘性。 |
| `penaltyEnabled` | `true` | 是否开启真实调用失败后的渠道惩罚。 |
| `penaltyFailureThreshold` | `2` | 同一 IP 连续失败多少次后惩罚同一上游渠道。 |
| `penaltySeconds` | `60` | 渠道进入惩罚中后，多久后尝试恢复检测。 |
| `healthCheckIntervalSeconds` | `30` | 可健康检测的渠道多久检测一次。健康检测规则不可关闭。 |
| `speedRankPenalty` | `300` | 渠道速度排名每落后一名增加多少熵值。 |
| `stickyHitPenalty` | `500` | 每多一个粘住的 IP，渠道或 Key 增加多少熵值。 |
| `forceAvailableButtonEnabled` | `true` | 后台是否显示/允许使用强制可用按钮。 |
